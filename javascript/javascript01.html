01. 변수 : 데이터 저장

변수는 프로그래밍에서 데이터를 저장하고 참조하는 데 사용되는 기본적인 요소로, 이름이 지정된 메모리 위치에 데이터를 저장할 수 있습니다.
변수를 사용함으로써 데이터를 효율적으로 관리하고 조작할 수 있습니다. 

{
    let x = 100;
    let y = 200;
    let z = "javascript";
    console.log(x);
    console.log(y);
    console.log(z);                
}
결과 확인하기
100
200
javascript

02. 변수 : 데이터 저장 + 데이터 변경

변수를 선언하고 값을 할당하여 데이터를 저장한 이후에도, 해당 변수에 새 값을 재할당하여 데이터를 추가 변경할 수 있습니다. 

{
    let x = 100;
    let y = 200;
    let z = "javascript";

    x = 300;
    y = 400;
    z = "jquery";

    console.log(x);
    console.log(y);
    console.log(z);  
}
결과 확인하기
300
400
jquery

05. 배열 : 데이터 저장(여러개) : 표현방법 1 : 배열 선언

배열은 여러 개의 값을 하나의 변수에 저장하는 자료 구조 입니다. Array 객체(JavaScript에서 사용되는 내장 객체로, 여러 개의 값을 순서대로 저장할 수 있는 컬렉션)를 사용하여 선언하고 동일한 종류의 데이터를 순서대로 저장하고 접근할 수 있습니다. 배열을 사용하여 여러 값을 저장하면 프로그램에서 해당 값들을 효율적으로 관리하고 사용할 수 있으며, 필요에 따라 값을 변경하거나 조작할 수도 있습니다. 

{
    const arr = new Array();

    arr[0] = 100;
    arr[1] = 200;
    arr[2] = "javascript";

    console.log(arr[0]);
    console.log(arr[1]);
    console.log(arr[2]);
}
결과 확인하기
100
200
javascript

배열을 선언하고 괄호안에 초기값을 나열함과 동시에 배열을 초기화 할 수 있습니다. 이러한 초기화 문법을 사용하면 코드를 간결하게 유지하고 배열을 쉽게 생성할 수 있습니다. 배열의 크기는 초기값의 개수에 따라 자동으로 결정됩니다. 

{
    const arr = new Array(100, 200, "javascript");

    console.log(arr[0]);

    console.log(arr[2]);
}

200

07. 배열 : 데이터 저장(여러개) : 표현방법 3 : 리터럴 방식으로 배열 선언

리터럴 방식으로 배열을 선언하는 것은 각 언어에서 배열의 초기값을 직접 지정하여 배열을 생성하는 방법을 의미합니다. 다양한 프로그래밍 언어에서 지원되며, 초기값을 괄호 안에 나열하여 배열을 선언합니다.
- 리터럴 방식으로 배열 선언시
<간결성> 리터럴 방식은 초기값을 직접 배열에 포함시키므로 코드가 간결해집니다. 따라서 배열을 선언하고 초기화하는 데 필요한 코드 양이 줄어듭니다.
<가독성> 배열의 초기값이 코드에 직접 나열되어 있으므로, 배열이 어떤 요소들을 포함하고 있는지 쉽게 파악할 수 있습니다. 이는 코드의 가독성을 높여줍니다.
<실시간 업데이트> 리터럴 방식으로 배열을 선언하면 해당 배열이 사용되는 부분에서 바로 바로 확인할 수 있습니다. 즉, 코드의 어디에서든 배열의 구조와 초기값을 확인할 수 있습니다.
<프로그래밍 언어의 표준화> 많은 프로그래밍 언어에서 리터럴 방식을 지원하고 있습니다. 이는 코드를 이해하기 쉽고 유지 보수하기 쉽게 만들어줍니다.
<코드 작성 시간 감소> 초기값을 직접 배열에 포함시키는 방식은 일반적으로 코드 작성 시간을 단축시켜줍니다. 배열의 요소를 하나씩 추가하는 대신 한 줄로 모든 초기값을 지정할 수 있습니다. 

{
    const arr = [];

    arr[0] = 100;
    arr[1] = 200;
    arr[2] = "javascript";

    console.log(arr[0]);
    console.log(arr[1]);
    console.log(arr[2]);
}
결과 확인하기
100
200
javascript

"push" 메서드는 JavaScript에서 배열에 새로운 요소를 추가하는 데 사용됩니다. 이 메서드를 사용하면 배열의 끝에 새로운 요소를 추가할 수 있습니다.

10. 배열 : 데이터 저장(여러개) : 표현방법 6 : 다차원 배열

다차원 배열은 배열의 요소로 또 다른 배열을 포함하는 배열입니다. 이는 배열 내부에 배열이 중첩되어 있는 형태를 말합니다. JavaScript에서 다차원 배열은 데이터를 구조화하고 복잡한 패턴을 표현하는 데 유용합니다. 예를 들어, 행렬(matrix)이나 이차원 그리드 형태의 데이터를 다룰 때 매우 효과적입니다.

11. 배열 : 데이터 저장(여러개) : 표현방법 7 : 배열구조분해할당

배열 구조 분해 할당(Array Destructuring Assignment)은 JavaScript에서 배열을 분해하여 개별 변수에 할당하는 문법입니다. 이를 통해 배열의 요소를 더 효율적으로 추출하고 사용할 수 있습니다. 

{        
    const arr = [100, 200, "javascript"];
    const [a, b, c] = arr;        

    console.log(a); // 100
    console.log(b); // 200
    console.log(c); // javascript        
}
결과 확인하기
100
200
javascript

12. 배열 : 데이터 저장(여러개) : 표현방법 8 : 배열펼침연산자

배열 펼침 연산자(Spread Operator)는 JavaScript에서 배열을 분해하여 각 요소를 개별적으로 처리할 수 있게 해주는 문법입니다. 이 연산자는 주로 배열을 다른 배열에 병합하거나 함수 호출 시 인수를 전달하는 데 사용됩니다. 

 {
    const arr1 = [100, 200];
    const arr2 = [300, 400];
    const merge1 = [...arr1, ...arr2];
    const merge2 = [...arr1, "javascript", ...arr2, "jquery"];

    console.log(arr1);      // [100, 200]
    console.log(...arr1);      // 100 200
    console.log(...arr1, ...arr2);  // 100 200 300 400
    console.log(merge1)   // [100, 200, 300, 400]
    console.log(merge2)   // [100, 200, 'javascript', 300, 400, 'jquery']
}
결과 확인하기
[100, 200]
100 200
100 200 300 400
[100, 200, 300, 400]
[100, 200, 'javascript', 300, 400, 'jquery']

JavaScript에서 배열 안에 객체를 포함하는 방식은 매우 일반적이며 유용한 패턴입니다. 이를 통해 복잡한 데이터를 구조화하고 관리할 수 있습니다. 각 객체는 중괄호{}로 정의되어 있고 각 객체는 키와 값으로 구성되어 있습니다.

}

객체 안에 배열을 포함하는 방식은 JavaScript에서 데이터를 구조화하고 표현하는 데 사용되는 일반적인 패턴 중 하나입니다. 이러한 구조는 객체의 속성 중 하나가 배열인 경우에 유용하게 활용됩니다. 배열을 사용하여 여러 값을 그룹화하고 관련된 데이터를 저장할 수 있습니다. 

{
    const obj = {
    a: 100,
    b: [200, 300],
    c: "javascript"
    }

    console.log(obj.a)   // 100
    console.log(obj.b[0])   // 200
    console.log(obj.b[1])   // 300
    console.log(obj.c)   // javascript                
}

19. 배열 : 데이터 저장(키와값) : 표현방법 7 : 객체 안에 객체가 있는 방식

객체 안에 객체를 포함하는 방식은 JavaScript에서 데이터를 구조화하고 표현하는 데 사용되는 일반적인 패턴 중 하나입니다. 이러한 구조는 객체의 속성 중 하나가 또 다른 객체인 경우에 유용하게 활용됩니다. 객체 안에 객체를 포함하는 방식은 중첩된 데이터 구조를 생성하고 관련된 데이터를 더 잘 정리할 수 있게 해줍니다.

20. 배열 : 데이터 저장(키와값) : 표현방법 8 : 객체 + 함수가 있는 방식

JavaScript에서 배열 안에 객체와 함수가 있는 방식은 객체와 함수를 포함하는 배열을 생성하는 방법을 말합니다. 이는 배열의 각 요소가 객체 또는 함수인 경우를 나타냅니다. 이러한 구조는 데이터와 관련된 로직을 효율적으로 구성하고 관리할 수 있게 해줍니다.

{
    const obj = {
    a: 100,
    b: [200, 300],
    c: { x: 400, y: 500 },
    d: "javascript",
    e: function () {
        console.log("javascript가 실행되었습니다.");
    },

    f: function () {
        console.log(obj.d + "가 실행되었습니다.");
    },

    g: function () {
        console.log(this.d + "가 실행되었습니다.");
    },
    }

    console.log(obj.a);
    console.log(obj.b[0]);
    console.log(obj.b[1]);
    console.log(obj.c.x);
    console.log(obj.c.y);
    console.log(obj.d);
    obj.e();
    obj.f();
    obj.g();
}

21. 배열 : 데이터 저장(키와값) : 표현방법 9 : 객체구조분해할당

객체 구조 분해 할당(Object Destructuring Assignment)은 JavaScript에서 객체의 속성을 추출하여 변수에 할당하는 방법입니다. 이를 통해 객체의 속성을 간편하게 사용할 수 있습니다. 객체 구조 분해 할당은 중괄호{}를 사용하여 객체의 특정 속성을 분해하고 해당 속성을 변수에 할당하는 방법입니다. 

 {
        const obj = {
        a: 100,
        b: 200,
        c: "javascript"
        }
        const { a, b, c } = obj;

        console.log(a);
        console.log(b);
        console.log(c);                
}

22. 객체 : 데이터 저장(키와값) : 표현방법10 : 객체 펼침연산자

객체 펼침 연산자(Object Spread Operator)는 JavaScript의 문법 중 하나로, 객체를 확장하거나 병합하는 데 사용됩니다. 이 연산자는 주로 객체의 속성을 복사하거나 새로운 속성을 추가할 때 유용하게 사용됩니다. 

{
    const obj1 = { a: 100, b: 200, }
    const obj2 = { c: 300, d: 400, }
    const merge1 = { ...obj1, ...obj2 }
    const merge2 = { ...obj1, x: "javascript", ...obj2, y: "jquery" }

    console.log(obj1);  // { a: 100, b: 200 }
    console.log(merge1); // { a: 100, b: 200, c: 300, d: 400 }
    console.log(merge2); // { a: 100, b: 200, x: "javascript", c: 300, d: 400, y: "jquery"}                               
}
결과 확인하기
{ a: 100, b: 200 }
{ a: 100, b: 200, c: 300, d: 400 }
{ a: 100, b: 200, x: "javascript", c: 300, d: 400, y: "jquery"}
